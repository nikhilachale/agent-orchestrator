The [Cursor Agent CLI](https://cursor.com/docs) brings Cursor's editing model to a terminal. AO spawns it per session, wraps `gh`/`git`, and tracks activity via the AO activity log.

<PlatformSupport macos="full" linux="full" windows="full" />

## Install

```bash
curl https://cursor.com/install -fsS | bash
```

Verify:

```bash
cursor-agent --help
```

You should see "Cursor Agent" in the output.

## Use

```yaml title="agent-orchestrator.yaml"
agent: cursor
```

No plugin-level config.

## How it works

- **Launch:** AO creates a Cursor chat with `cursor-agent create-chat`, stores the returned chat id, then runs `cursor-agent --resume <chatId> -- <prompt>` inside the worktree.
- **Activity tracking:** AO installs workspace-local Cursor hooks that report activity and can confirm the native chat id when Cursor emits it.
- **Session resume:** AO restores with `cursor-agent --resume <chatId>`. Deterministic restore is available for Cursor sessions created after AO stored a chat id; older Cursor sessions without `agentSessionId` cannot be restored safely and remain terminated.

## Detection

The plugin resolves `cursor-agent` from PATH or common install locations.

## Environment variables

| Variable          | Set by AO | Purpose                                 |
| ----------------- | --------- | --------------------------------------- |
| `AO_SESSION_ID`   | ✓         | AO session id                           |
| `AO_ISSUE_ID`     | ✓         | Issue identifier                        |
| `AO_DATA_DIR`     | ✓         | AO data root for hooks                  |
| `CURSOR_DATA_DIR` | ✓         | Isolated Cursor profile under `~/.ao`   |

## Troubleshooting

<Accordions>
	<Accordion title="AO says Cursor Agent isn't detected">
		Run `cursor-agent --help`. If it is missing, reinstall Cursor Agent or add the install directory to PATH.
	</Accordion>
	<Accordion title="An older Cursor session does not restore">
		Only Cursor sessions created after AO stored a native chat id have deterministic restore. Sessions without
		`agentSessionId` are left terminated instead of attaching AO to the wrong Cursor transcript.
	</Accordion>
</Accordions>
